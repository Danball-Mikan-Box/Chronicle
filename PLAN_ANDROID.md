# Chronicle Android 対応計画

## 概要
デスクトップ版の Chronicle を Android (Dioxus Mobile) に対応させるための計画。
モバイル端末の限られた画面スペースとタッチ操作に最適化した UI/UX を提供する。

## モバイル UI 戦略

### 1. 画面レイアウトの再設計
デスクトップ版の 3ペイン（サイドバー / エディタ / プレビュー）はモバイルでは成立しないため、以下の構成に変更する。

- **画面遷移（スタック/タブ方式）**:
    - **プロジェクト/エクスプローラー画面**: 章や話の一覧。ここから編集したい話を選択。
    - **編集画面**: 執筆に集中する画面。ツールバーは簡略化。
    - **プレビュー画面**: 編集画面からトグル、またはタブで切り替え。
- **ナビゲーション**:
    - ドロワー（サイドメニュー）またはボトムナビゲーションを使用して、主要な機能（エクスプローラー、設定、統計）にアクセス。
    - 編集画面では「戻る」ボタンでエクスプローラーに帰る。

### 2. モバイル向け操作の最適化
- **書式バー**: キーボードの上に表示されるアクセサリバーとして実装（モバイルエディタの定番）。
- **タッチターゲット**: ボタンやリスト項目のサイズを大きくし、誤操作を防ぐ。
- **ジェスチャー**: スワイプによるプレビュー切替などの検討。

### 3. 技術的課題
- **ファイルシステム**: Android の Scoped Storage への対応。プロジェクトフォルダをどこに置くか。
- **ダイアログ**: `rfd` (Rust File Dialog) がモバイルで動作するか確認。必要なら Dioxus ネイティブのダイアログに差し替え。
- **IME / キーボード**: 仮想キーボードが表示された際のレイアウト崩れ防止。
